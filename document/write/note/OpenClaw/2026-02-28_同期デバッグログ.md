# OpenClaw write → iCloud 同期デバッグログ（2026-02-28）

## 🎯 背景
`note-sync.sh` を統合し、fswatch + rsync でリアルタイム同期を構築。
しかし、launchd 経由で正しく動作していない疑いがあったためデバッグを実施。

---

# ✅ 使用スクリプト（統合版）

```bash
#!/bin/zsh

SRC="$HOME/.openclaw/workspace/document/write/note/OpenClaw"
DST="/Users/tbtsub/Library/Mobile Documents/com~apple~CloudDocs/Obsidian/note/OpenClaw"

fswatch -o "$SRC" | while read; do
  sleep 1
  rsync -a --delete "$SRC/" "$DST/"
done
```

---

# ✅ 切り分け手順

### ① rsync単体確認

```bash
rsync -a --delete "$SRC/" "$DST/"
```

→ 正常動作
→ パス問題ではないと判断

---

### ② fswatch単体確認

```bash
fswatch "$SRC"
```

→ ファイル変更時にイベント出力を確認

---

### ③ 手動実行テスト

```bash
zsh ~/bin/note-sync.sh
```

別ターミナルから：

```bash
touch test.md
```

→ 同期成功

---

# ✅ 想定原因

- launchd 環境では PATH が異なる
- fswatch が見つからない可能性
- フルパス指定が必要なケースあり

---

# ✅ 対策（安定化案）

launchd 使用時はフルパス指定を推奨：

```bash
FSWATCH="/opt/homebrew/bin/fswatch"
RSYNC="/usr/bin/rsync"
```

---

# ✅ 結果

- 手動実行で正常動作確認
- 監視 → 約1秒以内にiCloudへ同期
- 片方向同期は安定

---

# 🧠 学び

- launchd は通常のシェル環境と異なる
- PATH依存はトラブルの元
- まずは単体テストで切り分けるのが最短
